## 1. 项目概述

本项目是一个 **BLE + 经典蓝牙 SPP 双模调试工具**，集成了两种蓝牙通信方式：

| 通信方式 | 模块 | 适用场景 |
|---------|------|---------|
| BLE（低功耗蓝牙） | `ble` + `app` | 智能手环、传感器、IoT 设备等低功耗场景 |
| 经典蓝牙 SPP（串口协议） | `spp` | 蓝牙串口模块、HC-05/06、数据透传等场景 |

**技术栈：** Kotlin + MVVM + ViewBinding + Coroutines + EventBus

**关键信息：**
- `applicationId`: `com.bhm.ble`
- `compileSdk` / `targetSdk`: 36
- `minSdk`: 24
- `versionName`: 2.7.0
- Java 17

## 3. 程序入口

### 3.1 启动 Activity

程序入口在 `app/src/main/AndroidManifest.xml` 中声明：

```xml
<activity
    android:name=".ui.TransportSelectActivity"
    android:exported="true">
    <intent-filter>
        <action android:name="android.intent.action.MAIN" />
        <category android:name="android.intent.category.LAUNCHER" />
    </intent-filter>
</activity>
```

**`TransportSelectActivity`** 是整个 App 的启动页面，让用户选择使用 BLE 还是经典蓝牙 SPP。

### 3.2 入口代码

```kotlin
// app/src/main/java/com/bhm/demo/ui/TransportSelectActivity.kt
class TransportSelectActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        viewBinding = ActivityTransportSelectBinding.inflate(layoutInflater)
        setContentView(viewBinding.root)

        // 选择 BLE → 进入 BLE 主页面
        viewBinding.btnEnterBle.setOnClickListener {
            startActivity(Intent(this, MainActivity::class.java))
        }

        // 选择 SPP → 进入经典蓝牙终端
        viewBinding.btnEnterSpp.setOnClickListener {
            startActivity(Intent(this, SppMainActivity::class.java))
        }
    }
}
```

> **注意：** `SppMainActivity` 是通过 import 别名引入的：`import de.kai_morich.simple_bluetooth_terminal.MainActivity as SppMainActivity`，即 SPP 模块的 `MainActivity`。

### 3.3 Application 类

```xml
<!-- AndroidManifest.xml -->
<application android:name="com.bhm.support.sdk.common.BaseApplication" ...>
```

使用 `support` 模块的 `BaseApplication`，负责 MultiDex 初始化、Timber 日志 planted。

### 4.2 BLE 路径详细跳转

#### TransportSelectActivity → MainActivity

用户点击"BLE"按钮，直接 `startActivity` 进入 BLE 主页面。

#### MainActivity 内部跳转

`MainActivity` 是 BLE 路径的核心页面，提供设备扫描、筛选、连接功能。

| 触发动作 | 目标页面 | 条件 | 传递数据 |
|---------|---------|------|---------|
| 点击设备行 / `btnPreview` | `ScanBroadcastActivity` | 无特殊条件 | `BleDevice`（广播数据预览） |
| 点击 `btnConnect`（列表，未连接） | `DetailOperateActivity` | 连接成功后自动跳转 | `BleDevice` + `disConnectWhileClose=false` |
| 点击 `btnConnect`（列表，已连接） | 无跳转 | 直接断开连接 | — |
| 点击 `btnOperate`（已连接） | `DetailOperateActivity` | 设备已连接 | `BleDevice` + `disConnectWhileClose=false` |
| 底部输入 MAC 地址 `btnConnect` | `DetailOperateActivity` | 连接成功后自动跳转 | `BleDevice` + `disConnectWhileClose=true` |

关键跳转代码：

```kotlin
// MainActivity.kt — 连接成功后打开 GATT 操作页
private fun openGattDetailPage(bleDevice: BleDevice?, disconnectGattWhenLeavingDetail: Boolean) {
    if (viewModel.isConnected(bleDevice)) {
        val intent = Intent(this@MainActivity, DetailOperateActivity::class.java)
        intent.putExtra("data", bleDevice)
        intent.putExtra("disConnectWhileClose", disconnectGattWhenLeavingDetail)
        startActivity(intent) { _, resultIntent ->
            if (resultIntent != null) {
                showLoading("断开中...")
                // 返回结果表示需要断开连接
            }
        }
    } else {
        Toast.makeText(application, "设备未连接", Toast.LENGTH_SHORT).show()
    }
}
```

#### DetailOperateActivity 内部跳转

| 触发动作 | 目标页面 | 说明 |
|---------|---------|------|
| 写数据对话框中 `btnManagePreset` | `PresetWriteCommandActivity` | 管理预设写指令，返回后刷新预设按钮 |
| 按返回键 | 返回 `MainActivity` | 根据 `disConnectWhileClose` 决定是否关闭 GATT |
| EventBus 收到断连消息 | 自动 `finish()` | `MainViewModel` 发送断连事件 |

#### `disConnectWhileClose` 参数含义

| 值 | 场景 | 返回时的行为 |
|----|------|-------------|
| `false` | 从扫描列表连接 | 仅移除特征回调，保持连接（可重新进入操作页） |
| `true` | 通过 MAC 地址直连 | 关闭 GATT 连接，释放资源 |

### 4.3 SPP 路径详细跳转

SPP 路径使用 **Fragment** 进行页面切换，而非 Activity：

```
SPP MainActivity (Fragment 容器)
  │
  ├── DevicesFragment (默认显示)
  │     列出已配对(bonded)的经典蓝牙设备
  │     点击设备 → 替换为 TerminalFragment
  │
  └── TerminalFragment (加入返回栈)
        串口终端：发送/接收数据
        返回键 → popBackStack 回到 DevicesFragment
```

Fragment 切换代码：

```kotlin
// DevicesFragment.kt
override fun onListItemClick(l: ListView, v: View, position: Int, id: Long) {
    val device = listItems[position - 1]
    val args = Bundle().apply { putString("device", device.address) }
    val fragment = TerminalFragment().apply { arguments = args }
    parentFragmentManager.beginTransaction()
        .replace(R.id.fragment, fragment, "terminal")
        .addToBackStack(null)
        .commit()
}
```

### 4.4 Activity 间数据传递

| Intent Extra Key | 类型 | 传递方向 | 用途 |
|-----------------|------|---------|------|

| `"disConnectWhileClose"` | `Boolean` | `MainActivity` → `DetailOperateActivity` | 离开时是否断开 GATT |

## 5. BLE 模块详解

### 5.1 模块总览

`ble` 模块是一个独立的 BLE 封装库（Maven 坐标：`com.github.buhuiming:BleCore:2.7.0`），封装了 Android 原生 `android.bluetooth.*` API，提供简洁的 Kotlin DSL 风格调用。

**核心设计：**
- `BleManager` 单例作为唯一公共入口
- 基于 Kotlin 协程（非 RxJava）
- 任务队列串行化 BLE 操作，避免 GATT busy 错误
- 回调使用 DSL 风格注册

### 5.2 核心类关系

```
BleManager (单例，公共入口)
    │
    ▼
BleRequestImp (内部实现，构建回调 + 委托)
    │
    ▼
BleConnectedDeviceManager (设备池管理，LRU 策略)
    │
    ▼  (每个设备一个实例)
BleConnectedDevice (extends BluetoothGattCallback)
    │
    ├── BleConnectRequest     (连接)
    ├── BleScanRequest         (扫描，全局单例)
    ├── BleNotifyRequest       (通知)
    ├── BleIndicateRequest     (指示)
    ├── BleWriteRequest        (写入)
    ├── BleReadRequest         (读取)
    ├── BleMtuRequest          (MTU 协商)
    ├── BleRssiRequest         (RSSI 读取)
    └── BleSetPriorityRequest  (连接优先级)
         │
         ▼
    BleTaskQueue (单线程协程任务队列)
         │
         ▼
    BleTask (单个任务，suspendCoroutine 桥接 GATT 回调)
```

### 5.3 BleManager — 公共 API

`BleManager` 是整个 BLE 库的唯一入口，通过 `BleManager.get()` 获取单例。

#### 初始化

```kotlin
// 在 Application.onCreate 中初始化
val options = BleOptions.builder()
    .setScanDeviceName("MyDevice")        // 扫描设备名过滤
    .setScanMillisTimeOut(10_000)         // 扫描超时 10s
    .setConnectRetryCountAndInterval(2, 1_000)  // 连接重试 2 次，间隔 1s
    .setMtu(100, autoSetMtu = true)       // MTU 100，连接后自动设置
    .setEnableLog(true)                   // 开启日志
    .build()

BleManager.get().init(application, options)
```

#### 主要公共方法

| 方法 | 功能 | 对应 Android API |
|------|------|-----------------|
| `init(context, options)` | 初始化 BLE 管理器 | `getSystemService(BLUETOOTH_SERVICE)` |
| `startScan { ... }` | 开始扫描 | `BluetoothLeScanner.startScan()` |
| `stopScan()` | 停止扫描 | `BluetoothLeScanner.stopScan()` |
| `connect(bleDevice) { ... }` | 连接设备 | `BluetoothDevice.connectGatt()` |
| `connect(address) { ... }` | 通过 MAC 地址连接 | `adapter.getRemoteDevice()` + `connectGatt()` |
| `startScanAndConnect(...)` | 扫描并连接 | 扫描 + 连接组合 |
| `disConnect(bleDevice)` | 主动断开 | `BluetoothGatt.disconnect()` |
| `close(bleDevice)` | 断开并释放资源 | `disconnect()` + `close()` |
| `notify(bleDevice, serviceUUID, notifyUUID) { ... }` | 开启通知 | `setCharacteristicNotification()` + `writeDescriptor()` |
| `stopNotify(...)` | 停止通知 | `setCharacteristicNotification(false)` |
| `indicate(...)` | 开启指示 | 同 notify，使用 `ENABLE_INDICATION_VALUE` |
| `writeData(bleDevice, serviceUUID, writeUUID, data) { ... }` | 写数据 | `writeCharacteristic()` |
| `writeQueueData(...)` | 队列写数据（OTA 友好） | `writeCharacteristic()` + 重试机制 |
| `readData(...)` | 读数据 | `readCharacteristic()` |
| `setMtu(bleDevice, mtu) { ... }` | 设置 MTU | `requestMtu()` |
| `readRssi(bleDevice) { ... }` | 读取 RSSI | `readRemoteRssi()` |
| `setConnectionPriority(bleDevice, priority)` | 设置连接优先级 | `requestConnectionPriority()` |
| `getBluetoothGatt(bleDevice)` | 获取 GATT 对象 | 返回 `BluetoothGatt` |
| `isConnected(bleDevice)` | 判断是否已连接 | `bluetoothManager.getConnectedDevices()` |

#### 状态检查方法

```kotlin
BleManager.get().isBleSupport()    // 设备是否支持 BLE
BleManager.get().isBleEnable()     // 蓝牙是否已开启
BleManager.get().isScanning()      // 是否正在扫描
BleManager.get().isConnected(bleDevice)  // 设备是否已连接
```

### 5.5 回调 DSL

所有回调采用 Kotlin DSL 风格，通过 lambda 注册：

#### 扫描回调

```kotlin
BleManager.get().startScan {
    onScanStart { /* 扫描开始 */ }
    onLeScan { bleDevice, count -> /* 每次扫描到设备（含重复） */ }
    onLeScanDuplicateRemoval { bleDevice, count -> /* 去重后的设备 */ }
    onScanFail { failType -> /* 扫描失败，查看 BleScanFailType */ }
    onScanComplete { list, dedupedList -> /* 扫描完成，返回结果列表 */ }
}
```

#### 连接回调

```kotlin
BleManager.get().connect(bleDevice) {
    onConnectStart { device -> /* 开始连接 */ }
    onConnectSuccess { device, gatt -> /* 连接成功（服务发现完成后触发） */ }
    onConnectFail { device, failType -> /* 连接失败 */ }
    onDisConnecting { isActive, device, gatt, status -> /* 正在断开 */ }
    onDisConnected { isActive, device, gatt, status -> /* 已断开 */ }
}
```

> **重要：** `onConnectSuccess` 在 **服务发现完成后** 才触发，此时可以直接读取 GATT 服务树。

#### 通知回调

```kotlin
BleManager.get().notify(bleDevice, SERVICE_UUID, NOTIFY_UUID) {
    onNotifySuccess { device, uuid -> /* 通知开启成功 */ }
    onCharacteristicChanged { device, uuid, data -> /* 收到通知数据 */ }
    onNotifyFail { device, uuid, error -> /* 通知开启失败 */ }
}
```

#### 写入回调

```kotlin
BleManager.get().writeData(bleDevice, SERVICE_UUID, WRITE_UUID, byteArray) {
    onWriteSuccess { device, current, total, justWrite -> /* 单包写入成功 */ }
    onWriteFail { device, current, total, error -> /* 单包写入失败 */ }
    onWriteComplete { device, allSuccess -> /* 全部写入完成 */ }
}
```

#### 读取回调

```kotlin
BleManager.get().readData(bleDevice, SERVICE_UUID, READ_UUID) {
    onReadSuccess { device, data -> /* 读取成功 */ }
    onReadFail { device, error -> /* 读取失败 */ }
}
```

### 5.6 任务队列机制

BLE 操作必须串行执行（同时多个操作会导致 GATT busy），项目通过任务队列实现：

#### 三种队列模式（`BleTaskQueueType`）

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| `Default` | 同一设备的所有操作共享一个队列 | 默认模式，完全串行，最安全 |
| `Operate` | 按操作类型分队列（读/写/通知各自独立） | 需要读写并行的场景 |
| `Independent` | 按操作类型 + 特征 UUID 分队列 | 最大并发，不同特征互不阻塞 |

#### 任务执行流程

```
调用 writeData()
    │
    ▼
构建 BleTask（block 内使用 suspendCoroutine 挂起）
    │
    ▼
加入 BleTaskQueue（单线程协程）
    │
    ▼  等待前一个任务完成
执行 task.doTask() → 调用 BluetoothGatt.writeCharacteristic()
    │
    ▼  GATT 回调触发
BleConnectedDevice.onCharacteristicWrite() → continuation.resume()
    │
    ▼
触发用户回调（主线程）
    │
    ▼
自动执行下一个任务
```

### 5.7 设备管理

#### BleDevice（数据模型）

```kotlin
data class BleDevice(
    val deviceInfo: BluetoothDevice?,   // 系统设备对象
    val deviceName: String?,            // 设备名称
    val deviceAddress: String?,         // MAC 地址
    val rssi: Int?,                     // 信号强度
    val timestampNanos: Long?,          // 扫描时间戳
```

## 6. SPP 经典蓝牙模块详解

### 6.1 模块总览

`spp` 模块基于 Kai Morich 的 simple-bluetooth-terminal 开源项目，实现了经典蓝牙 SPP（Serial Port Profile）串口通信功能。

**核心特点：**
- 使用 RFCOMM 协议通过 SPP UUID 建立连接
- 前台 Service 保活，支持后台通信
- 双队列缓冲机制，确保 UI 切换时不丢数据
- 绑定 Service + 直接监听器接口通信（非 LocalBroadcast）

### 6.2 核心类关系

```
TerminalFragment (UI 界面)
    │  implements ServiceConnection, SerialListener
    │  bindService()
    ▼
SerialService (前台服务)
    │  implements SerialListener
    │  持有 SerialSocket
    ▼
SerialSocket (Socket 连接)
    │  implements Runnable
    │  使用 BluetoothSocket + RFCOMM
    ▼
BluetoothDevice.createRfcommSocketToServiceRecord(SPP_UUID)
```

### 6.3 SerialSocket — SPP 连接核心

`SerialSocket` 封装了经典蓝牙 RFCOMM 连接，是 SPP 通信的底层实现。

#### SPP UUID

```kotlin
companion object {
    // 标准 SPP UUID
    private val BLUETOOTH_SPP: UUID =
        UUID.fromString("00001101-0000-1000-8000-00805F9B34FB")
}
```

#### 连接与数据接收

```kotlin
// SerialSocket.kt — 在单线程 Executor 上执行
override fun run() {
    try {
        // 1. 创建 RFCOMM Socket
        socket = device.createRfcommSocketToServiceRecord(BLUETOOTH_SPP)
        // 2. 阻塞式连接
        socket?.connect()
        // 3. 通知连接成功
        listener?.onSerialConnect()
    } catch (e: Exception) {
        listener?.onSerialConnectError(e)
        socket?.close()
        return
    }
    connected = true
    // 4. 阻塞式读取循环
    try {
        val buffer = ByteArray(1024)
        while (true) {
            val len = socket?.inputStream?.read(buffer) ?: break
            val data = Arrays.copyOf(buffer, len)
            listener?.onSerialRead(data)  // 转发数据块
        }
    } catch (e: Exception) {
        connected = false
        listener?.onSerialIoError(e)
        socket?.close()
    }
}
```

#### 发送数据

```kotlin
fun write(data: ByteArray) {
    if (!connected) throw IOException("not connected")
    socket?.outputStream?.write(data)
}
```

> **注意：** `SerialSocket` 的 `init` 块强制要求传入 `applicationContext`（非 Activity Context），防止 Activity 泄漏。

### 6.4 SerialService — 前台服务

`SerialService` 是一个绑定式前台 Service，作为 `SerialSocket` 和 UI 之间的中间层。

#### 核心职责

1. **管理 Socket 连接**：持有 `SerialSocket` 引用
2. **线程切换**：将后台线程的回调转发到主线程
3. **数据合并**：合并高频数据块，减少 UI 刷新次数
4. **前台保活**：UI 不可见时显示通知，保持连接
5. **双队列缓冲**：UI 分离期间缓存事件，重连后回放

#### 公共 API

```kotlin
// 连接
fun connect(socket: SerialSocket)

// 断开
fun disconnect()

// 发送数据
fun write(data: ByteArray)

// UI 绑定/解绑
fun attach(listener: SerialListener)   // Fragment 可见时调用
fun detach()                           // Fragment 不可见时调用
```

#### 双队列缓冲机制

当 UI（`TerminalFragment`）分离时（如旋转屏幕、切到后台），Service 将事件缓存到两个队列：

- **queue1**：`detach()` 之前已 post 到主线程但尚未处理的事件
- **queue2**：`listener` 变为 null 后新到达的事件

当 UI 重新 `attach()` 时，依次回放 queue1 和 queue2 中的事件，确保不丢数据。

#### 数据合并优化

```kotlin
// SerialService.kt — 合并高频数据块
override fun onSerialRead(data: ByteArray) {
    synchronized(this) {
        if (listener != null) {
            synchronized(lastRead) {
                val first = lastRead.datas?.isEmpty() == true
                lastRead.add(data)  // 累积数据
            }
            if (first) {
                mainLooper.post {
                    // 一次性发送合并后的数据批次
                    listener?.onSerialRead(datas)
                }
            }
        } else {
            // UI 分离时缓存到 queue2
            queue2.last.add(data)
        }
    }
}
```

### 6.5 SerialListener — 回调接口

```kotlin
interface SerialListener {
    fun onSerialConnect()                         // 连接成功
    fun onSerialConnectError(e: Exception)        // 连接失败
    fun onSerialRead(data: ByteArray)             // 单块数据（Socket→Service）
    fun onSerialRead(datas: ArrayDeque<ByteArray>) // 合并批次（Service→UI）
    fun onSerialIoError(e: Exception)             // IO 错误
}
```

两个 `onSerialRead` 重载区分了原始单块回调（Socket→Service）和合并批次回调（Service→UI）。

```kotlin
    for (data in datas) {

        } else {
            // 文本模式：直接显示
```

### 8.3 BLE vs SPP 对比

| 特性 | BLE | 经典蓝牙 SPP |
|------|-----|-------------|
| 协议 | GATT over ATT | RFCOMM |
| UUID | 服务/特征各 UUID | 固定 SPP UUID |
| 连接方式 | `connectGatt()` | `createRfcommSocketToServiceRecord()` |
| 数据模型 | 特征值（Characteristic） | 流（InputStream/OutputStream） |
| 通信模式 | 读/写/通知/指示 | 双向流读写 |
| 功耗 | 低功耗 | 较高功耗 |

| 后台保活 | 不需要前台服务 | 需要前台 Service |
| Android API | `android.bluetooth.le.*` + `BluetoothGatt` | `BluetoothSocket` |

```kotlin
// 第 5 步：写入数据
BleManager.get().writeData(bleDevice, SERVICE_UUID, WRITE_UUID, byteArray) {
    onWriteComplete { device, allSuccess ->
        // 写入完成
    }
}

BleManager.get().close(bleDevice)
```

### 10.3 关键注意事项

1. **权限版本适配**：Android 12+ 使用 `BLUETOOTH_SCAN`/`BLUETOOTH_CONNECT`，以下使用定位权限
2. **BLE 操作串行化**：同时执行多个 GATT 操作会导致 busy，库已通过任务队列解决
3. **SPP 需要 applicationContext**：`SerialSocket` 强制要求非 Activity Context，防止泄漏
4. **SPP 前台服务**：Android 14+ 需要声明 `foregroundServiceType`
5. **连接成功时机**：BLE 的 `onConnectSuccess` 在服务发现完成后触发，可直接读取服务树
6. **MTU 与分包**：写入数据时单包不超过 `mtu - 3` 字节，使用 `BleUtil.subpackage()` 分包
7. **OptionSettingActivity 未被调用**：该 Activity 已注册但当前代码中无入口，属遗留代码
